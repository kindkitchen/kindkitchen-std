High-level expectations

- Use mapped import specifiers (JSR import map style) for std libraries instead of hard-coded deno.land URLs.
  - Example: import * as path from "@std/path" not import "https://deno.land/std/...".
  - If a std package is not yet mapped in the package's deno.json, add it with deno add --jsr, e.g.:
    - deno add --jsr @std/expect@^1.0.19
    - deno add --jsr @std/path@^1.1.4
  - Update the package/llm deno.json import map when adding new @std/* mappings.

Naming conventions

- Prefer snake_case for internal/local identifiers in code (functions, local variables, const names):
  - e.g., split_and_trim, include_glob_regex_list, default_skips
- Keep public/exported API names stable (do not rename exported functions or change their signature unless requested).
